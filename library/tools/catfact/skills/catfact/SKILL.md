# catfact

Generated by Gutenberg. Use this skill to call the catfact API through the local `catfact` CLI.

## When to use

- The user asks for data exposed by catfact (Breeds, Facts).
- An agent task needs structured JSON from https://catfact.ninja.
- You need to perform a write/destructive operation on catfact — always preview first, run with `--yes` only on explicit user confirmation.

## How to invoke

The skill assumes the generated CLI is on PATH (or invoke via `scripts/use-go.sh run ./cmd/catfact` from the project directory).

```bash
catfact operations
catfact call getBreeds --json
catfact sync --json
catfact search "<keyword>" --json
```


## Authentication

Set CATFACT_API_KEY in the environment, or pass `--api-key <key>`.

## Operations index

- `getBreeds` (GET /breeds) — read — Get a list of breeds
- `getRandomFact` (GET /fact) — read — Get Random Fact
- `getFacts` (GET /facts) — read — Get a list of facts


## Output contract

- All commands accept `--json` and emit machine-readable output.
- `call` returns `{ dryRun, operation, request, response }`. `dryRun: true` means the call was not executed (write op without `--yes`).
- `search` returns cached records. `sync` populates the local SQLite cache.
