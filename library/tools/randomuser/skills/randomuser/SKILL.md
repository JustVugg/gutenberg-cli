# randomuser

Generated by Gutenberg. Use this skill to call the randomuser API through the local `randomuser` CLI.

## When to use

- The user asks for data exposed by randomuser (api).
- An agent task needs structured JSON from https://randomuser.me.
- You need to perform a write/destructive operation on randomuser — always preview first, run with `--yes` only on explicit user confirmation.

## How to invoke

The skill assumes the generated CLI is on PATH (or invoke via `scripts/use-go.sh run ./cmd/randomuser` from the project directory).

```bash
randomuser operations
randomuser call getRoot --json
randomuser sync --json
randomuser search "<keyword>" --json
```


## Authentication

No authentication required.

## Operations index

- `getRoot` (GET /api) — read — GET /api


## Output contract

- All commands accept `--json` and emit machine-readable output.
- `call` returns `{ dryRun, operation, request, response }`. `dryRun: true` means the call was not executed (write op without `--yes`).
- `search` returns cached records. `sync` populates the local SQLite cache.
