# open-meteo

Generated by Gutenberg. Use this skill to call the open-meteo API through the local `open-meteo` CLI.

## When to use

- The user asks for data exposed by open-meteo (v1).
- An agent task needs structured JSON from https://api.open-meteo.com.
- You need to perform a write/destructive operation on open-meteo — always preview first, run with `--yes` only on explicit user confirmation.

## How to invoke

The skill assumes the generated CLI is on PATH (or invoke via `scripts/use-go.sh run ./cmd/open-meteo` from the project directory).

```bash
open-meteo operations
open-meteo call getForecast --json
open-meteo sync --json
open-meteo search "<keyword>" --json
```


## Authentication

No authentication required.

## Operations index

- `getForecast` (GET /v1/forecast) — read — GET /v1/forecast


## Output contract

- All commands accept `--json` and emit machine-readable output.
- `call` returns `{ dryRun, operation, request, response }`. `dryRun: true` means the call was not executed (write op without `--yes`).
- `search` returns cached records. `sync` populates the local SQLite cache.
